# A/B-тест банковского приложения 📱

**Оптимизация функционала банковского приложения: оценка влияния нового интерфейса на конверсию и вовлечённость пользователей**

> Проект Яндекс Лицея, специализация «Анализ данных» — 🥇 1 место в проектной части

---

## 🎯 Задача

Банк внедрил новый мобильный интерфейс. Нужно проверить, улучшает ли он ключевые метрики, и построить модель прогноза ежедневной посещаемости для планирования ресурсов.

---

## 📊 Данные

| Файл | Описание |
|------|----------|
| `users_ab.csv` | Характеристики ~5 000 пользователей |
| `events_ab.csv` | Событийные логи сессий |
| `visits_daily.csv` | Ежедневная посещаемость за 6 месяцев |

---

## 🔬 Методология

**A/B-тест:**
- 4 метрики на пользователя: `sessions_count`, `events_per_session`, `conversion_rate`, `session_duration`
- Статистические тесты: Mann-Whitney U, Z-test, χ²-test
- Сегментация по возрасту, устройству, городу

**Прогноз посещаемости:**
- OLS с календарными факторами (день недели, месяц, праздники)
- Сравнение: Linear Regression vs Polynomial Regression vs Random Forest

---

## 📈 Результаты

### A/B-тест

| Метрика | Группа A | Группа B | Значимость |
|---------|----------|----------|------------|
| Среднее число сессий | 6.05 | **7.98** | p < 0.001 |
| Длительность сессии | 4.89 мин | **9.43 мин** | p < 0.001 |
| Событий на сессию | 9.48 | 9.52 | p = 0.062 |
| Конверсия в транзакцию | 77.03% | 77.94% | p = 0.054 |

**Вывод:** новый интерфейс увеличивает вовлечённость (+32% сессий, +93% длительность), но не даёт значимого роста конверсии. Эффект однороден по сегментам.

### Прогноз посещаемости

| Модель | MAE | MAPE | RMSE |
|--------|-----|------|------|
| **Linear Regression** | **1 040** | **8.98%** | **1 579** |
| Polynomial Regression | 1 101 | 9.48% | 1 648 |
| Random Forest | 1 264 | 10.84% | 2 024 |

Линейная регрессия выбрана как основная модель. Посещаемость определяется календарными факторами; праздничные дни оказывают значимое отрицательное влияние (p < 0.01).
